# Лабораторная работа 3

Тема: Реализация собственных коллекций в C# (.NET).

## Цель

Познакомиться с интерфейсами коллекций в C#, реализовать простые коллекции и понять их контракт и функциональность.

## Реализованные коллекции

### 1. SimpleList<T>
- Реализует интерфейсы: `IEnumerable<T>`, `ICollection<T>`, `IList<T>`
- Внутренняя структура: динамический массив
- Поддерживает все основные операции: добавление, удаление, поиск, доступ по индексу
- Автоматически увеличивает емкость при необходимости

### 2. BidirectionalLinkedList<T>
- Реализует интерфейс `IList<T>`
- Двунаправленный связный список
- Поддерживает эффективные операции вставки/удаления в начале и конце

### 3. SimpleDictionary<TKey, TValue>
- Реализует интерфейсы: `IDictionary<TKey, TValue>`, `IReadOnlyDictionary<TKey, TValue>`
- Внутренняя структура: хеш-таблица с цепочками для разрешения коллизий
- Автоматическое изменение размера при достижении коэффициента загрузки

## Особенности реализации

### SimpleList
- Динамическое увеличение массива в 2 раза при заполнении
- Поддержка итерации через `foreach`
- Проверка границ индексов
- Отслеживание изменений коллекции для безопасности итераторов

### BidirectionalLinkedList
- Двунаправленные ссылки для эффективной навигации
- Оптимизированный поиск индекса (поиск с ближайшего конца)
- Память распределяется динамически под каждый элемент

### SimpleDictionary
- Разрешение коллизий методом цепочек
- Использование простых чисел для размера хеш-таблицы
- Коэффициент загрузки: 0.75
- Поддержка пользовательского компаратора

## Тестирование

Все коллекции покрыты модульными тестами с использованием NUnit. Тесты проверяют:
- Корректность основных операций (добавление, удаление, поиск)
- Обработку граничных случаев
- Поведение итераторов
- Потокобезопасность итераторов (бросают исключение при изменении коллекции)
